# CLAUDE.md

## Collaboration Rules

When working in this repository:

- Complete only the requested step
- Keep changes small and easy to review
- Preserve the existing structure unless there is a clear reason to adjust it
- Do not refactor unrelated files
- Keep the app runnable after each change

## Response Expectations

After implementation, explain:

1. What was changed
2. Why each new file or directory was necessary
3. How the resulting structure should be understood at a practical level

Prefer concise explanations. Expand only when the user asks for more depth.

## Safety Rules

- Do not delete working code unless it is clearly unnecessary
- Do not introduce large dependencies without explicit need
- Do not implement future product features ahead of schedule

## UI Collaboration

- Treat `DESIGN.md` as a design direction reference
- Keep early-stage pages intentionally simple
- Add visual complexity only when the step requires it

## Internationalization Collaboration

- Remember that the project will eventually support `zh-CN` and `ja`
- For early setup steps, prioritize maintainable structure over premature i18n implementation
- When i18n work begins, add both locales in the same step
